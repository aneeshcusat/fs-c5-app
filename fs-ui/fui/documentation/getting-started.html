Accessing the System

The Project Management System can be accessed through your web browser. You'll need your username and password, or you may use Microsoft Single Sign-On (SSO) if enabled for your organization.

Standard Login Process

Step 1: Open the Application
Navigate to the application URL provided by your administrator. The login page will be displayed automatically.
Step 2: Enter Credentials
  • Enter your username in the username field
  • Enter your password in the password field
  • Click the eye icon to show/hide your password if needed
Step 3: Sign In
Click the "Sign In" button to authenticate. The system will validate your credentials and redirect you to your default dashboard.
Step 4: Dashboard Access
After successful login, you'll be redirected to your default dashboard based on your access permissions:
  • Project Dashboard: If you have project management access
  • HRMS Dashboard: If you have HRMS access
  • Sales Dashboard: If you have sales module access
Login Page - Enter your username and password

Microsoft Single Sign-On (SSO)

If your organization has enabled Microsoft SSO:

Click the "Sign in with Microsoft" button on the login page
You'll be redirected to Microsoft's login page
Enter your Microsoft account credentials
Grant permissions if prompted
You'll be automatically logged into the system
üìå Note: SSO availability depends on your organization's configuration. If you don't see the Microsoft login option, use standard login.

Forgot Password

If you've forgotten your password:

Click the "Forgot Password" link on the login page
Enter your username or email address
Click "Submit" to receive password reset instructions
Check your email for reset instructions
Follow the link in the email to reset your password
‚ö†Ô∏è Warning: Password reset links expire after a certain time. If the link has expired, request a new one.

Account Activation

New users may need to activate their account:

You'll receive an activation email with a unique activation link
Click the activation link in the email
You'll be redirected to the account activation page
Set your new password (if required)
Complete any required profile information
Click "Activate Account" to complete the process

Session Management

Session Timeout: For security, your session will automatically expire after a period of inactivity. You'll need to log in again if your session expires.
Multiple Sessions: You can be logged in from multiple devices simultaneously, but be aware of potential conflicts when editing the same data.
Logout: Always click the "Sign Out" button (top right ‚Üí user icon ‚Üí Sign out) when finished, especially on shared computers.

üìö Basic Concepts

Understanding the System Structure

Projects

Definition: A project is a container for all work related to a specific client engagement, internal initiative, or business objective.
Key Attributes:
  • Project Name
  • Team/Department
  • Start and End Dates
  • Status (Active, Completed, Archived)
  • Account/Client Information

Tasks (Deliverables)

Definition: Tasks are specific work items within a project. They can be deliverables, milestones, or activities.
Key Attributes:
  • Task Name
  • Assigned Team Members
  • Due Dates
  • Status (Planned, In Progress, Completed, Closed)
  • Estimated vs Actual Hours

Work Items

Definition: Work items are the smallest unit of work, typically representing daily activities or time entries within a task.
Key Attributes:
  • Description of work
  • Time spent
  • Date
  • Related task

Teams

Definition: Teams are organizational units that group employees together. Projects are typically assigned to teams.

Accounts

Definition: Accounts represent clients or business entities. Projects are associated with accounts.

Status Types

Status Meaning When Used
Active Project/Task is currently in progress Default status for new items
Archived Project/Task is completed or inactive When work is finished but you want to keep records
Deleted Item is marked for deletion Can be undone if marked by mistake
Overdue Past due date Automatically applied when dates pass

Permission Levels

Your access to features depends on your role and permissions:

View

Can see and read information but cannot modify

Create

Can create new items

Edit

Can modify existing items

Delete

Can delete items (may require additional confirmation)

Archive

Can archive/unarchive items

Admin

Full access including configuration

üìå Note: If you don't see certain buttons or options, it's likely due to permission restrictions. Contact your administrator if you need additional access.

üîß Troubleshooting Common Issues

Login Issues

Cannot Log In

Check Credentials: Ensure username and password are correct (check caps lock)
Clear Browser Cache: Clear cookies and cache, then try again
Try Different Browser: Some browsers may have compatibility issues
Check Internet Connection: Ensure you have a stable connection
Contact Administrator: If issues persist, your account may need to be reset

Session Expired

If you see a "Session Expired" message:

  • Simply log in again
  • Your work is typically saved automatically
  • Consider saving work more frequently if sessions expire often

Display Issues

Page Not Loading Properly

Refresh the page (F5 or Ctrl+R / Cmd+R)
Clear browser cache and cookies
Try a different browser
Check if JavaScript is enabled

Sidebar Not Showing

If the left sidebar is hidden:

  • Click the hamburger menu icon (‚ò∞) in the top left
  • On mobile devices, the sidebar may be hidden by default - tap the menu icon

Feature Not Available

If you can't see a feature or button:

  • Check if you have the required permissions (contact administrator)
  • Verify the feature is enabled for your organization
  • Ensure you're on the correct page
  • Some features may be module-specific

Data Not Appearing

Empty Lists

Check your filters - you may have filtered out all items
Clear all filters and try again
Check date range filters
Verify you're looking at the correct team/project

Performance Issues

Slow Loading

  • Check your internet connection speed
  • Reduce the date range in filters
  • Limit the number of items displayed
  • Close other browser tabs/applications
  • Clear browser cache

Getting Help

If you continue to experience issues:

  1. Check this documentation for specific feature help
  2. Contact your system administrator
  3. Reach out to your team lead
  4. Use the Help button (?) in the application
